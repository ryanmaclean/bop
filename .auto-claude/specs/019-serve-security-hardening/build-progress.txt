Security Vulnerabilities Being Fixed:
1. Timing attack vulnerability - auth token comparison reveals token length/content via timing
2. Unauthenticated server - BOP_SERVE_TOKEN is optional, server runs without auth by default
3. DoS via huge payloads - no body size limit allows unbounded memory consumption
4. Path traversal - card ID validation misses encoded separators (%2F, %5C), .., null bytes
5. Missing security headers - responses lack X-Content-Type-Options, X-Frame-Options, Cache-Control
6. No rate limiting - endpoint can be flooded by attackers

Risk Level: HIGH

- Requires constant-time comparison to prevent timing attacks
- Must validate all attack vectors with unit tests
- Changes affect production HTTP server endpoint

Acceptance Criteria:

✓ Body > 64 KiB returns 413 Payload Too Large

✓ Security headers present on all responses
