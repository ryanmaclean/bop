Bug Description:
The ollama-local.nu adapter (lines 66-111) only writes output/result.md when the model
emits structured JSON with {"files": [...]}. For plain text responses (the common case),
stdout is captured but output/result.md is never written, causing the E2E test to fail.

Fix Location:
After line 111 in adapters/ollama-local.nu, add:
```nushell
# Fallback: if no structured JSON output, write stdout as result.md

    mkdir $"($workdir)/output"
    open --raw $stdout_abs | save --force $"($workdir)/output/result.md"
}
```
