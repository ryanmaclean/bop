# Full Pipeline Template — Example Spec

> This template runs all four stages: spec → plan → implement → qa.
> Use for complex or ambiguous work where you want each stage
> handled by a different agent with fresh context.

## Overview
Describe what needs to be built at a high level.
The spec agent will refine this into a precise specification.

## Rough Requirements
- What problem does this solve?
- Who is the user?
- What are the constraints?

## Scope Hints
- IN: areas to explore
- OUT: areas to avoid

## Acceptance Criteria
- [ ] All tests pass
- [ ] Lint and format clean
- [ ] QA agent signs off in `output/result.md`
